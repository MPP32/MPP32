# Security Policy

## Supported versions

| Version | Supported |
|---------|-----------|
| 1.x     | Yes       |
| < 1.0   | No        |

## Reporting a vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Send a description of the issue to **security@mpp32.org**. Include:

- A clear description of the vulnerability and its potential impact
- Steps to reproduce or a proof-of-concept (if safe to share)
- Any affected versions you have identified

We will acknowledge receipt within 48 hours and aim to release a fix within 14 days for confirmed issues.

## Scope

**In scope:**
- Authentication and authorization bypass
- Injection vulnerabilities (SQL, command, etc.)
- Sensitive data exposure
- Payment logic integrity issues

**Out of scope:**
- Social engineering attacks
- Physical security
- Third-party service vulnerabilities (DexScreener, Jupiter, CoinGecko)
- Denial-of-service attacks

## Disclosure

We follow a 90-day coordinated disclosure policy. We will credit reporters in release notes unless you prefer to remain anonymous.
