# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in MPP32, please report it responsibly.

**Do not open a public issue.** Instead, email **security@mpp32.org** with:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will acknowledge receipt within 48 hours and aim to issue a patch within 7 days for critical issues.

## Scope

The following are in scope:
- mpp32.org web application
- MPP32 API endpoints (`/api/*`)
- Payment proxy infrastructure
- Provider registration and management

## Out of scope

- Third-party services integrated by providers
- Social engineering attacks
- Denial of service attacks

## Responsible disclosure

We ask that you:
- Give us reasonable time to address the issue before public disclosure
- Do not access or modify other users' data
- Do not degrade the service for other users

We appreciate security researchers who help keep MPP32 safe.
